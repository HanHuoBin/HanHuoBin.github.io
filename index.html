Android性能优化之一：ViewStub

介绍

ViewStub 是一个轻量级的View，没有尺寸，不绘制任何东西，因此绘制或者移除时更省时。(ViewStub不可见，大小为0)

优点

实现View的延迟加载，避免资源的浪费，减少渲染时间，在需要的时候才加载View

缺点

  • ViewStub所要替代的layout文件中不能有标签
  • ViewStub在加载完后会被移除，或者说是被加载进来的layout替换掉了

用法

此处加个id为tv_extend的TextView是为了演示用的，点击显示和隐藏ViewStub，类似很多评论区的展开和收缩

布局

<?xml version=”1.0” encoding=”utf-8”?>

<ImageView
    android:id="@+id/img_logo"
    android:src="@mipmap/background"
    android:layout_width="300dp"
    android:layout_height="300dp"
    android:layout_marginTop="@dimen/dp_40"
    android:scaleType="centerCrop" />

<TextView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:layout_marginTop="@dimen/dp_10"
    android:gravity="center"
    android:text="hello,I'm TextView.\n I'm ImageView"/>


@Override
public void onClick(View v) {
switch (v.getId()) {
case R.id.tv_extend:
if(extendTv.getText().equals(“展开”)){
viewStub.setVisibility(View.VISIBLE);
extendTv.setText(“关闭”);
}else{
viewStub.setVisibility(View.GONE);
extendTv.setText(“展开”);
}
break;
}
}

git .ignore不起作用

原因

.ignore只能忽略那些原来没有被纳入版本控制的文件，如果文件已经存在版本控制中，则修改 .gitignore是无效的，解决办法是先把本地缓存删除，然后提交，重新纳入版本控制系统中

Android onClicker防止重复点击

Veröffentlicht am 2016-02-08

Android onClicker防止重复点击

场景

相信大家经常遇到这种问题，快速点击按钮会多次请求接口（一方面因为应用或手机卡顿，另一方便很多应用没有设置按钮点击时按钮变化，导致用户误以为没有点击按钮，哈哈哈，当然我们的测试会经常提类似的bug），添加的时候会出现多条一样的数据。

解决方案

那么遇到这种情况我们如何来避免重复点击呢！

  • 比如，我们可以设置点击事件在规定时间内只响应一次
    • 这个也有多种实现方式，比如在每次点击之后添加一个时间判断，超过时间段执行操作，两次点击间隔在时间段内不执行操作
    • 比如重写OnClickListener方法，在onClickListener内做时间判断，两次点击在时间段内不分发点击事件，反之返回
  • 比如按钮在点击之后就设置为不可点击，直到点击按钮对应的任务完全执行完之后再设置按钮为可点击状态

我这边主要要说的实现方案是重写 OnClickListener

优势

这么写有个优势，不需要修改原代码的逻辑，只需要把OnClickListener替换成NoMultipleClickListener即可。不需要每次都去做判断，主需要关注业务逻辑就行了，代码看上去也会更加简洁
